# 自动删除无效用户插件

## 插件简介

定时清理无效用户和过期用户，支持试运行模式和安全保护。

## 配置项说明

| 配置项 | 说明 | 默认值 | 类型 |
| --- | --- | --- | --- |
| `enable_auto_delete` | 启用自动删除功能，关闭时为试运行模式 | `false` | boolean |
| `schedule_frequency` | 定时任务频率（minutely/hourly/daily） | `hourly` | select |
| `delete_days` | 注册超过该天数仍未激活的用户才会被清理 | `7` | number |
| `delete_expired_users_after_days` | 删除过期超过指定天数的用户 | `0` | number |
| `batch_size` | 每批次最多处理的用户数量 | `1024` | number |
| `max_batches_per_run` | 每次调度最多连续处理的批次数 | `10` | number |

**定时任务频率说明：**
- `minutely`：每分钟执行一次
- `hourly`：每小时执行一次
- `daily`：每天00:00执行一次

**删除过期用户说明：**
- `0`：不删除过期用户（使用基础清理模式）
- `>0`：删除过期超过指定天数的用户

## 清理模式

### 基础模式（默认）
清理注册超过指定天数的无效用户：

**查询条件：**
- `is_admin = false` - 非管理员用户
- `plan_id IS NULL` - 无套餐
- `transfer_enable = 0` - 无流量
- `expired_at = 0 OR expired_at IS NULL` - 未设置到期时间，兼容旧版和新版注册数据
- `balance = 0` - 余额为0
- `commission_balance = 0` - 佣金余额为0
- 无邀请关系 - 未邀请过其他用户
- `created_at < (当前时间 - 注册天数阈值)` - 注册超过指定天数

### 过期用户模式
清理过期超过指定天数的用户：

**查询条件：**
- `is_admin = false` - 非管理员用户
- `expired_at > 0` - 已设置过期时间
- `expired_at < (当前时间 - 过期天数阈值)` - 过期超过指定天数
- `balance = 0` - 余额为0
- `commission_balance = 0` - 佣金余额为0
- `created_at < (当前时间 - 注册天数阈值)` - 注册超过指定天数

## 使用建议

1. **首次启用**：建议先设置 `enable_auto_delete = false`（试运行模式），通过日志确认清理范围后再开启真实删除
2. **清理模式选择**：
   - 基础清理：适合清理纯被刷注册的用户
   - 过期用户清理：适合清理过期n天未续费的用户
3. **性能调节**：根据服务器性能调节"单次处理数量"，避免数据库压力过大
4. **监控日志**：定期审阅任务日志，了解清理效果并及时调整阈值
5. **安全保障**：插件保护管理员、有余额和有佣金的用户，两种清理模式均不会删除管理员

## 大用户量性能优化

当 `v2_user` 表达到数万或数十万用户时，基础模式需要判断“用户是否从未邀请过其他用户”。该判断依赖：

```sql
invited.invite_user_id = v2_user.id
```

如果 `invite_user_id` 没有索引，候选用户查询会非常慢，即使 `batch_size` 已经调大，也可能长时间卡在查找候选阶段。建议在大用户量环境手动添加以下索引：

```sql
ALTER TABLE v2_user
    ADD INDEX idx_cleanup_invite_user_id_id (invite_user_id, id);
```

添加前可先检查是否已存在：

```sql
SHOW INDEX FROM v2_user;
```

注意事项：

- 该索引不是插件自动创建的，需要数据库管理员按需手动执行。
- 大表加索引会消耗 MySQL IO/CPU，建议在低峰期执行。
- 如果正在运行清理任务，建议先临时关闭或降低清理频率，避免慢查询占用元数据锁。
- 该索引主要优化基础模式下的“无邀请关系”判断，对大量无效注册用户的清理速度提升明显。

## 技术特性

- **并发安全**：使用缓存锁防止任务重复执行
- **试运行优化**：仅在试运行模式统计待删数量，真实删除时直接按批次拉取候选用户
- **事务保护**：每批用户删除操作都是原子性的
- **批量处理**：支持批量删除用户、订单、邀请码、统计和工单数据
- **游标扫描**：按用户 ID 顺序记录扫描游标，降低重复扫描成本
- **孤儿数据清理**：自动清理旧版删除工单后遗留的孤儿工单消息
- **错误恢复**：单批用户删除失败不影响后续调度继续执行
- **详细日志**：提供完整的执行过程记录
